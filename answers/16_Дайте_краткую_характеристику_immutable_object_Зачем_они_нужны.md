Immutable object (неизменяемый объект) - это объект, состояние которого не может быть изменено после его создания. Если объект неизменяемый, то все его поля, которые являются ссылками на другие объекты, также неизменяемые.

Неизменяемые объекты полезны во многих ситуациях, особенно в многопоточных средах. Такие объекты могут использоваться в качестве ключей в HashMap или HashSet, поскольку их значение не может быть изменено после создания. Также неизменяемые объекты обеспечивают безопасность при работе в многопоточной среде, поскольку они не могут быть изменены из разных потоков, что устраняет необходимость в синхронизации.

В Java многие стандартные классы, такие как String, Integer, Long, Boolean и другие, являются неизменяемыми. Кроме того, в Java можно создавать собственные неизменяемые классы, которые могут использоваться в приложениях.
